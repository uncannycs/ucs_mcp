API Key Form — All Fields

Fig 4.1 — API Key form: Identity, Rate Limiting, Permissions, Model Access Control sections + Chatter

Section: IDENTITY

Key NameRequired. Displayed in audit log and list view. tracking=True — changes appear in chatter.
OwnerLinked Odoo user. Tool calls run in this user's env — their record rules and ACLs apply. Cascade-deletes key when user is removed.
Key PrefixAuto-set on creation. First 12 chars of plain key (e.g. ucs_mcp_eiv8). Read-only — shown for identification only.
Expiry DateOptional. Key rejected after this date with AccessError. Blank = no expiry.
ActiveUncheck to deactivate without deleting. Inactive key gets AccessError on all calls.

Section: RATE LIMITING

Rate Limit (req/hr)Max requests per 1-hour rolling window. Default 100. 0 = unlimited.
Requests This HourRead-only counter. Resets automatically when window expires (>3600s elapsed).
Rate Window StartRead-only. When current 1-hour window began.
Last UsedRead-only. Datetime of most recent authenticated call.
Total UsesRead-only. Cumulative count across all windows.

Section: PERMISSIONS

Read PermissionDefault ON. Required by: search_records, get_record, get_fields, search_count, smart_context, check_access, and all informational tools.
Write PermissionDefault OFF. Required by: write_record, send_message, schedule_activity.
Create PermissionDefault OFF. Required by: create_record.
Delete PermissionDefault OFF. Required by: delete_record.
Execute PermissionDefault OFF. Required by: execute_method (14 dangerous method names still blocked regardless).

Section: MODEL ACCESS CONTROL

Allowed ModelsComma-separated model names. If non-empty, key can only access these. Example: sale.order,account.move,res.partner,product.product
Custom Blocked ModelsAdditional per-key blocked models on top of system defaults. Example: hr.payslip, res.users

System Blocked (always blocked, cannot override):
res.users, res.groups, ir.config_parameter, ir.rule, ir.model.access, base_setup.config.settings, res.users.apikeys, auth.totp.device, mail.message, mail.followers, bus.bus, ir.cron

Chatter / History

Tracked fields: Name, Owner, Expiry, Active, Rate Limit, Write/Create/Delete/Execute. "MCP API Key created" message added on creation. Shows all field changes with timestamps.

Creating a New API Key

Fig 5.1 — New API Key form (empty) — all fields visible before first save
Navigate: MCP → API Keys → New
Enter Key Name — e.g. "Production Claude Code"
Set Owner — defaults to current user. Key runs tools in this user's security context.
Set Rate Limit — 100/hr default. 0 = unlimited. Good starting point: 200/hr for AI clients.
Check permissions — Read is on by default. Enable Write/Create/Delete/Execute only as needed.
Restrict models (optional) — Allowed Models narrows access; Custom Blocked adds protection beyond system defaults.
Save — plain key shown once in dialog. Copy it immediately — it is never stored and cannot be retrieved again.

Key format: ucs_mcp_<44-char URL-safe random>. Stored as SHA-256 hex digest only. Plain key shown once at creation.

MCP Protocol Usage

Endpoint: POST /mcp/v1 — JSON-RPC 2.0. Auth via Authorization: Bearer <key> header.

Initialize (no auth)

POST /mcp/v1
Content-Type: application/json

{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}

Response:
{"jsonrpc":"2.0","id":1,"result":{"protocolVersion":"2024-11-05","serverInfo":{"name":"ucs-mcp-odoo","version":"19.0.1.0.0"},"capabilities":{"tools":{}}}}

List Tools

POST /mcp/v1
Authorization: Bearer ucs_mcp_YOUR_KEY_HERE
{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}

Response: {"result":{"tools":[{"name":"search_records","description":"...","inputSchema":{...}}, ...]}}

Call a Tool

POST /mcp/v1
Authorization: Bearer ucs_mcp_YOUR_KEY_HERE

{"jsonrpc":"2.0","id":3,"method":"tools/call",
 "params":{"name":"search_records","arguments":{"model":"sale.order","domain":[["state","=","sale"]],"limit":10}}}

Response:
{"result":{"content":[{"type":"text","text":"{"count":5,"records":[...]}"}]}}

Security Model

Five-layer check on every authenticated API call:

1. AuthSHA-256 hash lookup. Checks active flag, expiry date, rate limit window.
2. PermissionKey must have the required permission (read/write/create/delete/execute) for the tool.
3. System Blocked12 sensitive models always blocked regardless of key config — cannot be unblocked.
4. Custom BlockedPer-key custom_blocked_models adds extra blocked models for this key specifically.
5. Allowed ListIf allowed_models is non-empty, model must be in that set or access is denied.

Env isolation: All tool execution runs as the key owner: request.env(user=api_key.user_id.id). Odoo record rules, model ACLs, and company isolation all apply exactly as if the owner were logged in directly.

Dangerous Methods (blocked in execute_method regardless of execute permission):
write, create, unlink, _write, _create, execute, execute_kw, sudo, with_context, with_env, with_user, with_company, _sql, env
