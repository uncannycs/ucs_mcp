Setup

From install to first AI answer in three steps

No custom code. No middleware. No third-party accounts. Just install the module, generate a key, and paste one URL into your AI tool.

01

Install the module

Install UCS MCP Server from the Odoo Apps list. No dependencies beyond base Odoo — works on Community and Enterprise.

02

Generate an API key

Visit /mcp/connect in your browser. Name the key, choose permissions (read-only or write access), copy the URL.

03

Connect your AI tool

Paste the URL into Claude Code, ChatGPT Apps, Cursor, or any MCP client. Your AI now sees your Odoo data live.

Example: "Show me all overdue invoices for customers in Dubai" — Claude Code queries Odoo live and returns a formatted table. No copy-pasting, no exports, no waiting for a report.

The self-service portal at /mcp/connect — users create and manage their own API keys without admin help.

Compatibility

Works with the AI tools your team already uses

One Odoo module, one URL — compatible with every major AI client that supports the Model Context Protocol.

Easiest
💬
ChatGPT
Paste the URL directly in ChatGPT Apps → New App. No config file, no terminal.
chatgpt.com → Apps → New App → Server URL

Claude Code
One terminal command connects permanently. Works in any directory.
claude mcp add --transport http odoo "..."

Cursor IDE
Add to .cursor/mcp.json. Restart Cursor — Odoo appears as a tool in the AI panel.
.cursor/mcp.json → mcpServers block
🤖
Claude Desktop
Edit claude_desktop_config.json with the provided snippet. Requires Node.js for the bridge.
npx mcp-remote + config JSON
🦊
Gemini CLI
Add to ~/.gemini/settings.json. Works with gemini CLI from Google.
~/.gemini/settings.json → mcpServers
🔨
Continue.dev
VS Code and JetBrains extension. Odoo data available inside your code editor.
~/.continue/config.json → mcpServers
💻
Codex CLI
OpenAI's terminal agent. Connects via ~/.codex/config.yaml.
~/.codex/config.yaml → mcp_servers
🐍
OpenAI Python SDK
Python bridge script loads all 25 Odoo tools into any GPT-4o or o1 call. Works with localhost too.

Any MCP Client
Standard MCP protocol (JSON-RPC 2.0 over HTTP). Any client that follows the spec will work.

Security & Control

Your data, your rules

Every key is scoped, logged and revocable. The AI can only do what you explicitly allow — nothing more.

🔑

SHA-256 Key Auth

Plain keys are never stored — only the hash. Even if the database is compromised, keys cannot be recovered.

✅

Per-key Permissions

Each key has independent Read / Edit / Create / Execute / Delete permissions. Grant only what the AI needs.

📋

Full Audit Log

Every tool call is logged — key name, user, tool used, response status, duration, and IP address. Nothing is hidden.

🚫

Model Blocklist

Sensitive system models (user passwords, security rules, module config) are permanently blocked — AI cannot touch them even with full access.

⏱

Rate Limiting

Set a request cap per key per hour. Prevents accidental runaway loops from consuming your server resources.

⏳

Expiry Dates

Set keys to expire after 30, 90 or 365 days. Expired keys stop working automatically — no manual cleanup needed.

The built-in audit dashboard shows every AI tool call — who used it, when, and whether it succeeded.
